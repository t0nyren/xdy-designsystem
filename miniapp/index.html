H · Mini-Program 点餐小程序

把「现包」
装进点餐的每一步

微信点餐小程序 UI 设计。差异化不在下单快——在让顾客看见现包：门店卡上的「现包中」呼吸灯、菜单里的当日现包批次、下单后的擀皮-包制-蒸制实时进度。所有界面均按 DS v3.1 tokens 落地，以下五屏为可交互的高保真 HTML 稿（图中价格均为示例）。

01 · Principles 设计原则

P1三步下单

首页 → 点单 → 确认，任何商品从打开到下单不超过三屏；购物栏常驻底部拇指热区。

P2现包可视化

「现包中」状态贯穿全程：门店卡呼吸灯 → 商品的现包批次标签 → 订单页的四步现包进度。是品牌资产，不是功能点缀。

P3纪律照搬 DS

面粉白纸底、暖墨字、绛红只做主行动与热点（≤5%）、金只做辅助标识；一屏一个红色主体，字阶与网页端同源。

03 · Components 组件规范

主行动按钮 / 购物栏

提交订单加入购物袋
  • 绛红 --seal #9F3B3B 只给一屏内唯一主行动；次级行动用暖墨 --ink。
  • 圆角 999px 胶囊形，高度 ≥ 42px（拇指热区 ≥ 44×44 含边距）。
  • 购物栏常驻底部，墨底金额白字，数量角标才允许第二处红。

状态标签

今日现包生饺可带走售罄
  • 「今日现包」是唯一允许用红的标签；其余标签一律纸二 --paper-2 底 + 金墨 --gold-ink 字。
  • 标签不叠加超过 2 个；售罄降灰 #6B6259 并禁用加号。

现包进度 Steps

擀皮备馅 已完成 · 金墨实心
现包中 进行中 · 绛红呼吸
  • 完成态金墨实心点，进行态绛红 + 1.6s 呼吸动画（同门店卡「现包中」）。
  • 文案写动作不写系统词：「案板上正在为你捏褶」而非「制作中」。

字阶与数字

¥16.9白菜猪肉水饺
  • 价格与取餐码用 Outfit --font-en，¥ 符号缩至 0.72em。
  • 商品名黑体 500；只有屏级标题用宋体 600——小程序里宋体是点缀不是正文。
  • 正文最小 12px（小程序 rpx 折算 24rpx），辅助信息 10.5px 起。

04 · Flow 点单流程

进入扫码 / 搜索 / 附近门店
→
